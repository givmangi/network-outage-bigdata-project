# Internet Outage & Service Quality Intelligence

A Big Data pipeline that ingests real-time and historical internet measurement data from **RIPE Atlas** and **IODA**, processes it through a Bronze → Silver → Gold medallion architecture, and exposes ISP performance analytics via Jupyter Notebooks.

**Stack:** Kafka · MinIO · Apache Spark · TimescaleDB · Jupyter

---

## Architecture

```
RIPE Atlas WebSocket  ──►  ripe-ingester  ──►  Kafka (raw.ripe.ping)
                                           ──►  MinIO  (bronze/ripe/ping/...)
                                                          │
IODA REST API         ──►  ingester        ──►  Kafka (raw.ioda.*)        │
                                           ──►  MinIO  (bronze/ioda/...)   │
                                                          │
                                                    Spark Silver jobs
                                                          │
                                                   MinIO (silver/...)
                                                          │
                                                    Spark Gold job
                                                          │
                                                   TimescaleDB (asn_baselines)
                                                          │
                                                   Jupyter Notebooks
```

---

## Prerequisites

- Docker + Docker Compose
- Python 3.10+ (host machine, for `backfill.py` only)
- `pip install requests python-dotenv` (host machine)

## 1. Configuration

Copy and fill in the `.env` file in the project root:

```env
KAFKA_CLUSTER_ID=<generate with: python -c "import uuid, base64; print(base64.urlsafe_b64encode(uuid.uuid4().bytes).decode().rstrip('='))">
MINIO_ROOT_USER=admin
MINIO_ROOT_PASSWORD=yourpassword
TARGET_COUNTRIES=IT MM IN PK UA RU PS SY IR TR BD NG US DE GB

# PostgreSQL / TimescaleDB Secrets
TIMESCALEDB_USER=admin
TIMESCALEDB_PASSWORD=yourpassword
```

---

## 2. Build the RIPE Probe Mapping (run once)

This fetches all active RIPE Atlas probe IDs for your target countries and saves them locally. The ingesters use this file to filter the global measurement stream and inject ASN data at the bronze layer.

```bash
cd ingestion/ripe
pip install requests python-dotenv
python ripe_recon.py
cd ../..
```

This produces `ingestion/ripe/ripe_probe_mapping.json`.

---

## 3. Start the Always-On Stack

```bash
docker compose up -d kafka minio minio-init ingester ripe-ingester spark-silver-stream timescaledb
```

This starts:
- **Kafka** — message transport between ingesters and Spark
- **MinIO** — S3-compatible object storage for bronze and silver layers
- **ingester** — polls IODA API, writes to Kafka + MinIO bronze
- **ripe-ingester** — subscribes to RIPE Atlas WebSocket, writes to Kafka + MinIO bronze
- **spark-silver-stream** — consumes Kafka live, writes silver Parquet to MinIO
- **timescaledb** — time-series database for the gold serving layer

Check everything is healthy:

```bash
docker compose ps
```

Optional UIs:
- MinIO console: http://localhost:9090
- Kafka UI: http://localhost:8080

---

## 4. Historical Backfill (optional but recommended)

To load historical data instead of waiting for live data to accumulate:

```bash
# Both RIPE and IODA, last 7 days
python backfill.py --source all --days 7

# RIPE only
python backfill.py --source ripe --days 7

# IODA only, specific countries
python backfill.py --source ioda --countries IT DE --days 7
```

---

## 5. Run the Silver Batch Jobs

Transforms raw bronze JSON/NDJSON into clean Parquet in the silver layer.

```bash
# RIPE silver — auto-discovers all available bronze partitions
docker compose run --rm spark-silver-ripe

# RIPE silver — specific date range
docker compose run --rm spark-silver-ripe --start 2026-06-01 --end 2026-06-20

# IODA silver
docker compose run --rm spark-silver-ioda
docker compose run --rm spark-silver-ioda --start 2026-06-01 --end 2026-06-20
```

---

## 6. Run the Gold Batch Job

Aggregates silver data into hourly baselines per ASN/country and writes to TimescaleDB.

First, make sure the `asn_baselines` table exists in TimescaleDB:

```bash
docker exec -it ioda-timescaledb psql -U admin -d outage_intelligence -c "
CREATE TABLE IF NOT EXISTS asn_baselines (
    time_window         TIMESTAMPTZ NOT NULL,
    country_code        TEXT        NOT NULL,
    asn                 INTEGER     NOT NULL,
    rtt_median_ms       DOUBLE PRECISION,
    loss_95th_pct       DOUBLE PRECISION,
    total_measurements  INTEGER
);
SELECT create_hypertable('asn_baselines', 'time_window', if_not_exists => TRUE);
"
```

Then run the gold job:

```bash
docker compose run --rm spark-gold
```

Verify rows landed:

```bash
docker exec -it ioda-timescaledb psql -U admin -d outage_intelligence -c "
SELECT country_code, COUNT(*) as rows FROM asn_baselines GROUP BY country_code ORDER BY country_code;
"
```

## 7. Jupyter Notebooks

Start Jupyter:

```bash
docker compose up -d jupyter
docker logs ioda-jupyter 2>&1 | grep "http://127"
```

Open the URL printed by the logs. Notebooks are saved in `./notebooks/` on your host.

Connect to TimescaleDB from inside a notebook:

```python
from sqlalchemy import create_engine
import pandas as pd

engine = create_engine(
    "postgresql://admin:yourpassword@timescaledb:5432/outage_intelligence"
)

df = pd.read_sql("""
    SELECT
        country_code,
        asn,
        ROUND(AVG(rtt_median_ms)::numeric, 2) AS avg_rtt_ms,
        ROUND(AVG(loss_95th_pct)::numeric, 4) AS avg_loss,
        SUM(total_measurements)               AS total_measurements
    FROM asn_baselines
    WHERE rtt_median_ms > 0
      AND loss_95th_pct < 0.5
    GROUP BY country_code, asn
    ORDER BY avg_rtt_ms
""", engine)

df.head(20)
```

## Re-running the Full Pipeline (e.g. after new data arrives)

```bash
# 1. Backfill fresh bronze data
python backfill.py --source ripe --days 3

# 2. Re-process silver
docker compose run --rm spark-silver-ripe

# 3. Re-aggregate gold
docker compose run --rm spark-gold
```
